When HTML5 has become a standard? What were the main reasons for developing new standard?

HTML5 became a standard in 2014, when the World Wide Web Consortium (W3C) finalized the latest iteration of the HTML specification. The main reasons for developing a new standard were to address the limitations of previous HTML versions and to improve the overall web experience for users and developers. Some main reasons for developing HTML5 are:

1. Better video support: HTML5 introduced the <video> and <audio> elements, allowing for native playback of multimedia content without relying on plugins like Flash.

2. Cleaner code: HTML5 introduced new semantic elements like <header>, <footer>, <nav>, and <article>, which provide more meaningful structure to web pages.

3. Improved user experience: HTML5 introduced features like the <canvas> element for dynamic graphics and animations, as well as support for local storage and offline applications.

4. Enhanced performance: HTML5 enabled faster page loading times and more efficient use of resources, resulting in improved overall performance.

5. Improved accessibility: HTML5 introduced new accessibility features, making it easier for people with disabilities to use the web.

6. Cross-platform compatibility: HTML5 was designed to work across different devices and platforms, ensuring a consistent user experience regardless of the device being used.

7. Developer-friendly: HTML5 simplified web development by providing a more consistent and structured approach, making it easier for developers to create complex web applications.

So, HTML5 has become the standard for web development due to its numerous improvements and enhancements over previous HTML versions.

What is a difference between W3C HTML5 and WHATWG HTML Living Standard?

The main differences between W3C HTML5 and WHATWG HTML Living Standard are:

1. Goals and approach: The W3C aimed to publish a "finished" version of HTML5, while the WHATWG focused on continuously evolving the HTML standard, hence the name "HTML Living Standard"

2. Authoring advice and requirements: The W3C HTML specification includes authoring advice and requirements that differ markedly from the WHATWG HTML Living Standard

3. Versioning: The W3C HTML5 specification is versioned, while the WHATWG HTML Living Standard is not. This means that the WHATWG specification is constantly updated and evolving, without specific version numbers

4. Collaboration: In 2019, W3C and WHATWG agreed to work together on HTML and DOM, producing a Living Standard and Recommendation/Review Draft-snapshots

The W3C HTML5 specification is a more traditional, versioned approach to standardizing HTML, while the WHATWG HTML Living Standard is a continuous, evolving standard that is updated more frequently.

Comment on semantic and presentational markup. Why presentational markup is considered deprecated now?

Semantic markup is the use of HTML elements according to their intended purpose, which reinforces the meaning and structure of the content. On the other hand, presentational markup involves using HTML elements to control the appearance of the content, such as using <font> or <big> tags to change the font size.

Presentational markup is considered deprecated now because it mixes content and presentation, making the code less maintainable and less accessible. By separating content and presentation, semantic markup allows for better accessibility, easier maintenance, and more consistent styling across different devices and browsers.

For example, instead of using the<b> tag to make text bold, semantic markup would use the <strong> tag to indicate strong importance. Similarly, instead of using the <font> tag to change font size, semantic markup would use CSS to control the font size and other styling properties.

In summary, semantic markup is preferred over presentational markup because it provides better structure, accessibility, and maintainability for web content.

What are the audio and video formats, officially supported by HTML standard? What happens if you use other format?

There are three supported video formats: MP4, WebM, and Ogg.

There are three supported audio formats: MP3, WAV, and OGG.

There could be problems in using other formats, there is a possibility that it will work wrong or won't work at all

In case if you need other formats in your project you have to use a third-party video or audio player that supports these formats. However same there could be problems, using non-standard formats may result problems in compatibility and performance across different browsers and devices.
